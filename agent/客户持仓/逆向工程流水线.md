# 总体原则（先读）

你写的 Prompt 必须满足 4 点：

1. **不让模型直接给“投资建议”**（避免套路化）
2. **强迫模型区分：资产 vs 人 vs 系统**
3. **要求“分阶段 / 分状态”而不是一次性回答**
4. **要求“为什么这样处理”，而不是“该不该卖”**

# 第一组 Prompt：客户资产出现亏损时，系统**整体**怎么处理？

## Prompt 1｜亏损后的第一性原则（非常关键）

```text
假设客户资产已经出现明显亏损。

请不要给任何具体投资建议，而是回答：
1. 一个金融决策系统在“客户亏损发生后”，最优先要保护的是什么？
2. 亏损发生后，系统关注的对象应按什么顺序排序？（资产 / 组合 / 行为 / 情绪）
3. 为什么“继续优化收益”在某些亏损阶段是危险的？

请用结构化要点回答，并说明背后的决策逻辑。
```

👉 **你在观察什么**

* 模型会不会立刻谈“长期收益”
* 有没有“目标函数切换”的意识
* 是否把“人”放进系统

---

## Prompt 2｜亏损阶段划分（逼模型暴露隐含状态机）

```text
请将“客户资产亏损”拆分为多个阶段，而不是一个状态。

例如（仅示意）：
- 轻度亏损
- 中度亏损
- 严重亏损

对每一个阶段，请回答：
1. 系统的核心目标是什么？
2. 哪些行为是最危险的？
3. 系统应当“多做什么 / 少做什么”？

请以表格形式输出。
```

👉 **你在观察什么**

* 模型是否天然使用“阶段 / 状态”
* 有没有把“行为风险”单独拎出来

---

## Prompt 3｜什么时候系统“应该闭嘴”

```text
在客户资产亏损的过程中：

1. 是否存在某些阶段，金融决策系统“应该减少建议甚至不建议”？
2. 为什么在这些阶段，更多的信息和建议反而可能伤害客户？
3. 一个成熟系统应如何“降低干预强度”而不是“提高建议密度”？

请从行为金融与系统风险角度回答。
```

👉 **这是识别“成熟模型”的分水岭问题**

# 第二组 Prompt：不同资产类别出现亏损时，系统如何区分处理？

> ⚠️ 这一组是你**最关键的逆向工程素材**

---

## Prompt 4｜资产类别 × 亏损处理逻辑（核心）

```text
请从“金融决策系统设计”的角度回答：

当以下不同资产类别出现亏损时，
系统是否应该采用不同的处理逻辑？为什么？

资产类别包括：
- 现金类
- 固定收益类
- 股票/权益类
- 另类资产

请对每一类资产分别说明：
1. 该资产“亏损”的典型含义是什么？
2. 该类资产在亏损时，最主要的风险来自哪里？
3. 系统在处理该类资产亏损时，最容易犯的错误是什么？
4. 系统应重点防范哪一类“错误决策”？

请用表格输出。
```

👉 **这是你后面构建“资产差异化规则”的底稿**

---

## Prompt 5｜同样是亏损，哪些“可以忍”，哪些“必须警惕”？

```text
请回答以下问题：

1. 在不同资产类别中，哪些类型的亏损在长期配置中是“可预期、可承受”的？
2. 哪些类型的亏损往往意味着“结构性问题”或“不可逆风险”？
3. 一个金融决策系统应如何区分这两类亏损？

请避免使用“应该卖出/买入”的表述，
而是从“系统信号识别”的角度回答。
```

👉 **你在提取：模型对“正常亏损 vs 异常亏损”的认知**

---

## Prompt 6｜资产亏损 × 行为风险联动（高级）

```text
请从行为金融的角度分析：

当不同资产类别出现亏损时，
投资者最常见的错误行为分别是什么？

请按资产类别分别说明：
- 典型情绪反应
- 常见错误决策
- 为什么这种资产更容易触发该行为

并说明：
金融决策系统应如何通过“设计”而非“说教”来降低这些行为风险。
```

👉 **这是“资产特性 × 行为护栏”的核心素材**

# 第三组 Prompt：把“处理方式”逼成“系统规则”

## Prompt 7｜从答案到规则（非常重要）

```text
基于你前面关于“资产亏损处理”的分析，

请将结论进一步抽象为：
- 可以被系统执行的【规则】
- 可以被系统识别的【风险信号】
- 必须被系统限制的【危险行为】

要求：
1. 不涉及具体资产或产品
2. 使用中性、工程化语言
3. 假设系统处于“银行代销，而非资产管理”前提下

请用表格输出。
```

👉 **这是你后面做“规则引擎 / 状态机”的原材料**

# 第四组 Prompt：横向对比（逆向工程杀器）

## Prompt 8｜多模型一致性测试（可选）

```text
请你重新回答“客户资产出现亏损时，系统应如何处理”，
但这一次，请你站在以下不同角色视角分别回答：

1. 风控负责人
2. 行为金融学者
3. 金融系统工程师
4. 客户保护与合规负责人

请比较这些视角下的共识与分歧。
```

👉 **你会看到：
真正稳定的系统逻辑，一定是“多角色收敛”的部分。**
